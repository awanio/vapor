The updated implementation uses a dropdown component placed before the search input, with status indicators for each option.

Implementation Details

Changes Made:

  • Created a new reusable filter-dropdown component in src/components/ui/filter-dropdown.ts
  • Updated ansible-executions.ts to import and use the new dropdown component
  • Replaced the filter buttons with a dropdown that includes all status options (All, Running, Successful, Failed, Canceled, Pending)
  • Positioned the dropdown before the search input for better visual hierarchy
  • Added status indicator dots to visually distinguish different statuses
  • Maintained the same filtering functionality with improved UI

Benefits:

  • More compact UI - saves horizontal space
  • Scalable - can easily add more filter options without cluttering the interface
  • Consistent with other dropdown patterns in the application
  • Visual status indicators make it easier to identify status types
  • Reusable component that can be used in other views
